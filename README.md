## 项目简介（项目名称：校园互助服务平台 - CampusHub）

### 1.项目背景
在高校日常生活中，学生之间的互助需求广泛且高频，典型场景包括：

* 快递代取：因上课、实习或外出等原因无法及时领取快递，希望同楼栋或同校园的同学代为领取并送达；

* 学习辅导：针对课程作业、考试复习、技能学习（如编程、设计、外语等）寻求同学帮助或组队学习；

* 二手物品交换：教材、笔记、电子产品、生活用品等的转让、赠送或借用；

* 活动组队：寻找一起参加比赛、社团活动、运动、旅行、自习等同伴。



目前，这些需求主要依赖微信群、QQ 群、贴吧、朋友圈等非结构化渠道发布，存在以下痛点：

* 信息碎片化：需求分散在不同群组，难以集中浏览和检索；

* 匹配效率低：缺乏按类型、地点、时间等维度的筛选与推荐机制；

* 信任缺失：缺少身份认证、评价体系，学生担心安全问题或交易纠纷；

* 责任模糊：无任务流程跟踪，无法确认是否完成、是否准时、是否有纠纷记录。

**CampusHub** 旨在构建一个统一、可信、高效的校园互助服务平台，帮助学生完成从“发布需求 → 智能匹配 → 沟通确认 → 完成协作 → 互评激励”的完整闭环，提升校园协作效率与学生群体间信任感。、


### 2.用户角色 

**需求方（发布互助需求的学生）**
* 浏览申请列表，选择服务方
* 确认任务完成
* 对服务方进行评价
* 查看自己的发布记录与信用分	小张下午有课，无法去快递点取包裹，他在CampusHub上发布“16:00前帮忙代取XX快递，送到7号宿舍楼，酬谢一杯奶茶”

**服务方（接单并提供服务的学生）**
* 申请承接任务
* 与需求方沟通确认细节
* 标记任务进度（如“已取件”“已送达”）
* 请求需求方确认完成
* 对需求方进行评价	小李正好下午没课，看到小张的快递代取需求且同宿舍楼，申请承接并在16:00前完成送达，获得好评和信用分

**平台管理员（平台管理维护人员）**
* 审核/下架违规需求（如校外广告、敏感内容）
* 处理用户投诉与纠纷仲裁
* 查看平台数据统计（需求总量、完成率、活跃度、接取广告发布情况等）
* 发布平台公告或信用奖惩规则	有用户发布“代考”违规需求，管理员收到举报后将其下架并警告该用户；两位同学因任务完成标准产生争执，管理员介入调解并记录仲裁结果

### 3.核心需求与重要功能模块
**模块⼀：⽤户管理**

注册/登录（⽀持学号+密码，或第三⽅登录、个⼈资料管理（头像、昵称、学院、联系⽅式、⻆⾊切换（发布需求人/接单服务人/管理员）

**模块⼆：需求发布与浏览**

发布互助需求（标题、描述、分类等）、需求分类体系、需求列表浏览（⽀持按分类、时间、距离筛选）、需求详情查看                                                                

**模块三：智能匹配与推荐**

基于分类、地理位置、时间的基础匹配、基于⽤户历史⾏为的简单推荐（如经常接快递代取的⼈优先推荐快递类需求）、匹配结果排序

**模块四：订单流程管理**

接单 → 确认 → 进⾏中 → 完成 → 评价 
、双⽅确认完成机制
、订单历史记录

**模块五：评价与信⽤体系**

完成订单后双向评分（1-5 星 + ⽂字评价）、
⽤户信⽤分计算与展示、
信⽤等级标识

**模块六：消息通知**

站内消息（接单通知、状态变更通知、评价通知）、
简易即时通讯（需求⽅与服务⽅的⼀对⼀消息）、

**模块七：管理后台**

⽤户管理（查看、禁⽤）、
需求内容审核、
数据统计看板

## 项目技术栈
### 整体架构：前后端分离

采用前后端分离架构，前端负责用户界面交互与数据展示，后端负责业务逻辑、数据存储与接口提供。双方通过 RESTful API 进行通信，提升开发效率与系统可维护性。

### 前端（Vue）

* 语言：Java17
* 框架：Vue 3.0x
* 构建工具：Vite
* 路由：Vue Router
* UI组件库： Element Plus等组件库
* HTTP请求：原生 fetch API + Axios
* SPA式架构，实现页面切换的流畅过渡

### 后端：Springboot

* 语言：Java17
* 框架：Spring Boot 3.0x
* API 风格：RESTFUL API
* 构建工具：Maven
* 数据库访问：MyBatis
* 数据库：MySQL
* 身份认证：Springboot Security

### 其他工具
* 版本控制：Git + GitLab
* 自动化测试：GitLab CI
* 接口管理：Postman
