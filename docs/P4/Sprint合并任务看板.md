# Sprint 合并任务看板规范

> **项目：** 校园互助服务平台（CampusHub）
> **阶段：** P4 — 编码与开发

---

## 一、概述

### 任务概览
| 优先级 | 模块 | 任务 | 估计工时 | 负责人 | 完成标准 |
|------|------|------|---------|-------|---------|

| P0 | 需求发布 | 发布需求、需求列表、需求详情 | 5h | 刘映池 | 需求发布、查看需求列表、搜索需求、查看需求详情功能能在开发环境的前端页面下使用，实现需求编辑页面、需求列表页面、需求详情页面，能够通过单元测试及与后续模块结合的集成测试 |
| P0 | 订单管理 | 接单、状态流转、订单历史 | 5h | 刘渝川 | 接单、状态流转、订单历史功能能在开发环境、测试环境、生产环境下使用，实现用户管理自己订单的前端界面和根据身份对订单进行操作，能够通过单元测试及与后续模块结合的集成测试 |
| P1 | 评价与信用体系 | 双向评分、信用分计算 | 5h | 孙博一 | 双向评分、信用分计算功能能在开发环境、测试环境、生产环境下使用，实现用户评价、信用分计算功能，实现评分与评价页面和其他页面（主页等）对信用分的展示与计算，能够通过单元测试及与后续模块结合的集成测试 |
| P1 | 消息通知 | 通知触发、消息模板、通知记录、用户交互 | 5h |  陆俊达|实现消息更新时的通知和用户间进行对话的会话页面实现与实时更新（类似微信聊天界面）|

| P3 | 数据与操作安全 | 通过后端对权限和数据进行控制，对数据进行加密存储，对用户操作进行审计，对异常行为进行拦截 | 10h | 暂定 | 确保系统在开发环境、测试环境、生产环境下能够安全运行，满足权限控制与数据保护需求 |

**先确保P0任务、P1任务开发结束，如果有空闲时间再对P2任务，P3任务进行开发。**



## 二、任务拆分

### 用户管理
| 任务 | 估计工时 | 依赖 |
|------|---------|------|
| 后端：用户注册接口（手机号/邮箱、密码加密） | 1h | 无 |
| 后端：用户登录接口（JWT生成与验证） | 1h | 注册接口 |
| 后端：个人资料CRUD接口（获取/更新/修改密码） | 1h | 登录认证 |
| 后端：用户信息前端持久化（LocalStorage/Token管理） | 0.5h | 登录接口 |
| 前端：注册页面设计与表单验证 | 0.5h | 无 |
| 前端：登录页面设计与Token存储 | 0.5h | 注册页面 |
| 前端：个人中心页面（资料展示/编辑/头像上传） | 0.5h | 资料接口 |
| 单元测试：用户Service层测试 | 1h | 接口完成 |


### 需求发布
| 任务 | 估计工时 | 依赖 |
|------|---------|------|

| 后端：需求列表接口（分页、排序、筛选） | 1h | 发布接口 |
| 后端：需求详情接口 | 0.5h | 列表接口 |
| 后端：搜索需求接口（关键词匹配） | 0.5h | 列表接口 |
| 后端：删除/修改需求接口（仅发布者可操作） | 0.5h | 发布接口 |
| 前端：需求编辑/发布页面（表单+富文本） | 1h | 发布接口 |
| 前端：需求列表页面（卡片式展示、分页、筛选栏） | 0.5h | 列表接口 |
| 前端：需求详情页面（信息展示+接单按钮） | 0.5h | 详情接口 |
| 单元测试：需求Service层测试 | 0.5h | 接口完成 |


### 订单管理
| 任务 | 估计工时 | 依赖 |
