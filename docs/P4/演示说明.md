## 环境说明与应用启动

### 1. 配置数据库并启动后端：

首先，确保设备上已安装 MySQL 数据库。

然后可以根据本地root用户的密码修改`SEC-II-2026/backend/src/main/resources/application.properties`文件中的数据库连接信息。[打开配置文件](../../backend/src/main/resources/application.properties)


```properties

spring.datasource.username=root
spring.datasource.password=ubuntu (修改为本地root用户的密码)

```



完成配置后，在项目根目录下运行以下命令启动后端服务（或者在IDE中点击按钮运行BackendApplication.java）：

```bash
npm  run dev:backend

```

启动成功后，后端服务将运行在 http://localhost:8080 端口。如果是首次运行，你应该会在命令台中看到类似下面的数据库初始化信息：

```

======== 数据初始化完成 ========
统计信息：
  - 用户数：***
  - 需求数：***
  - 订单数：***
Hibernate: 
    select
        count(*) 
    from
        reviews r1_0
  - 评分数：***
============================

```

或者你可以访问http://localhost:8080/hello，如果后端正常运行，你会在浏览器页面看到 `Hello, This is Spring Boot!` 的信息显示。


### 2. 启动前端：

首先，确保设备上已安装 Node.js。

然后在前端目录`SEC-II-2026/frontend`下运行命令下载前端依赖：

```bash
npm install

```

完成依赖下载后，运行以下命令启动前端服务：

```bash
npm run dev:frontend

```

启动成功后，前端服务将运行在 http://localhost:5173 端口。你可以在浏览器中访问 http://localhost:5173 查看前端页面以确定前端是否成功运行。

## 核心路径说明


### 1.登录/注册账号

前端登录和注册账号url的路径为：`/login`和`/register`。

- 在输入框中输入账号和密码，点击登录按钮，如果账号和密码正确，则会跳转到主页。如果账号和密码错误，则会显示错误提示。

- 在输入框中输入账号和密码，点击注册按钮，如果账号和密码正确，则会跳转到主页。如果注册失败，则会显示错误提示。

### 2.主页

前端主页路径为：`/`。

- 在主页中，你可以看到所有需求，点击需求卡片，可以查看需求详情。
- 你可以搜索框中输入需求关键词，可以搜索需求。
- 你可以根据需求类型筛选需求。
- 你也可以点击`发布需求`按钮，发布新的需求。
- 你也可以点击头像按钮，通过点击对应按钮选项，查看自己的用户页面/消息界面/管理后台（待开发）或者退出登录。

### 3.发布需求

前端发布需求路径为：`/create/demand`。

- 在发布需求页面中，你可以填写需求信息，点击发布按钮，发布新的需求。

### 4.需求详情

前端需求详情路径为：`/demand/:id`。

- 在需求详情页面中，你可以查看需求详情，点击`联系需求发布者`按钮，可以联系需求发布者。
- 你也可以点击`接取订单`按钮，将需求加入订单。

### 5.订单详情

前端订单详情路径为：`/order/:id`。

- 在订单详情页面中，你可以查看订单详情，如果是订单接取者，则可以点击`联系需求发布者`按钮，可以联系需求发布者。
- 如果是需求发布者，你可以点击`联系接单者`按钮，可以联系接单者。
- 根据身份不同，你可以点击`取消订单`/`开始服务`/`确认完成`按钮，将订单状态更新为完成。

### 6.用户信息页面

前端用户页面路径为：`/user/:id`。

- 在用户页面中，你可以查看该用户基本信息和收获的评价。

### 7.用户主页

前端用户主页路径为：`/myprofile`。

- 在用户主页中，你可以查看自己的基本信息。
- 你可以点击`修改昵称`按钮，修改昵称。
- 你可以点击`修改头像`按钮，修改头像。
- 你可以点击`我的需求`按钮，查看自己的需求。
- 你可以点击`我的订单`按钮，查看自己的订单。
- 你可以点击`我的消息`按钮，查看自己的消息。

### 8.用户发布的需求

前端显示用户发布的需求页面路径为：`/my/demands`。

- 在用户发布需求页面中，你可以查看自己发布的需求。
- 你可以点击需求卡片，查看需求详情。
- 你可以根据类别、状态、关键词等条件搜索发布的需求。

### 9.用户相关的订单

前端显示用户相关的订单页面路径为：`/my/orders`。

- 在用户相关的订单页面中，你可以查看自己相关的订单。
- 你可以点击订单卡片，查看订单详情。
- 你可以根据状态、角色搜索自己相关的订单。
- 你可以评论完成的订单。

### 10.用户对话页面

前端显示用户对话和收到的消息页面路径为：`/my/conversations`。

- 在用户收到消息页面中，你可以查看自己收到的消息。
- 你可以点击会话卡片，查看历史消息详情。


### 11.编辑自己发布的未被接取的需求界面

前端编辑自己发布的未被接取的需求界面路径为：`/edit/demand/:id`。

- 在编辑自己发布的未被接取的需求界面中，你可以修改需求信息。
